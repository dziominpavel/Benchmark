# Benchmark

Репозиторий для сравнения LLM-моделей на задачах из живых Android-проектов
(`GymProgress`, `VoiceMind`, `Wishlot` — Kotlin/Compose/Room).

Здесь нет кода приложений — только текстовые артефакты: задачи, снапшоты волн,
прогоны моделей, leaderboard. Код живёт в отдельных репозиториях по соседству
(`C:/projects/<ProjectName>`).

## Зачем

Подобрать LLM-модель под свои задачи, опираясь на воспроизводимые прогоны
в реальном коде, а не на маркетинговые бенчмарки. Модели соревнуются на одних
и тех же задачах, на одинаковом снапшоте кода, по единому чек-листу.

## Структура

```
Benchmark/
├── AGENTS.md                 # Протокол для модели-участника (что делать при прогоне)
├── README.md                 # Этот файл
├── models.yaml               # Реестр LLM-моделей (id, имя, провайдер, версия)
├── tasks/                    # Задачи 4 категорий
│   ├── _TEMPLATE.md          # Шаблон задачи
│   ├── _CHECKLIST.md         # Единый чек-лист оценки (0–5 по каждому пункту)
│   ├── feature/              # F-NNN-<slug>.md — новые фичи
│   ├── bugfix/               # B-NNN-<slug>.md — багфиксы по репорту
│   ├── refactor/             # R-NNN-<slug>.md — рефакторинг/архитектура
│   └── review/               # V-NNN-<slug>.md — code review / планирование
├── snapshots/                # Квартальные снапшоты проектов
│   ├── _TEMPLATE.md
│   └── wave-NN-YYYYQn/       # Одна папка на волну
│       ├── README.md         # Дата, что нового, ссылки на теги
│       └── <project>.yaml    # commit hash + git_tag + сводка по проекту
├── runs/                     # Прогоны моделей
│   ├── _TEMPLATE.md          # Шаблон прогона (YAML front matter + заметки)
│   └── wave-NN-YYYYQn/       # <task-id>__<model-id>.md
├── waves/                    # Итоги волн
│   └── wave-NN-YYYYQn/
│       └── summary.md        # Рейтинг моделей, выводы по волне
├── leaderboard.md            # Сводный рейтинг (генерируется tools/aggregate.py)
├── tools/
│   └── aggregate.py          # Читает runs/ → перегенерирует leaderboard.md
└── openspec/                 # Спецификации самой бенчмарк-системы (spec-driven)
```

## Как это работает

### Волны (раз в квартал)

Каждые ~3 месяца объявляется новая **волна** (`wave-NN-YYYYQn`):
- В каждом проекте ставится git-тег `benchmark-wave-NN` на текущий коммит.
- В `snapshots/<wave>/` фиксируется commit hash + тег + краткая сводка.
- Все модели в одной волне бегут на **одинаковом** коде → сравнимо.

### Прогон одной модели на одной задаче

1. Пользователь открывает сессию с выбранной моделью.
2. Модель читает `AGENTS.md` → задачу → правила проекта.
3. В папке проекта делает `git checkout benchmark-wave-NN`.
4. Выполняет задачу, проверяет сборку/тесты.
5. Создаёт файл прогона `runs/<wave>/<task-id>__<model-id>.md`
   с баллами по чек-листу и заметками.
6. Откатывает изменения в проекте (снапшот остаётся чистым).

### Leaderboard

`tools/aggregate.py` парсит YAML front matter из `runs/**/*.md` и перегенерирует
`leaderboard.md`: общий рейтинг + разбивка по категориям + по волнам.

## Категории задач

| Категория | Префикс | Что проверяет |
|-----------|---------|---------------|
| Feature | `F-NNN` | Способность довести фичу до рабочего кода по спеке |
| Bugfix | `B-NNN` | Дебаггинг: найти причину и починить по репорту |
| Refactor | `R-NNN` | Архитектурное мышление: рефакторинг, миграции |
| Review | `V-NNN` | Критическое мышление: code review / план без кода |

## OpenSpec

Сама бенчмарк-система описана спецификациями в `openspec/`:
- `task-format` — формат задач 4 категорий
- `run-protocol` — процедура прогона модели
- `wave-snapshot` — квартальное версионирование
- `leaderboard` — агрегация результатов

Изменения системы идут через OpenSpec workflow: `/opsx:propose` → `/opsx:apply` → `/opsx:archive`.
Артефакты на русском, normative keywords `MUST`/`SHALL`/`MUST NOT` — на английском.

## Быстрый старт

1. Добавить модель в `models.yaml`.
2. Создать задачу по шаблону `tasks/_TEMPLATE.md` в нужной категории.
3. Зафиксировать волну: поставить git-тег `benchmark-wave-NN` в проекте,
   заполнить `snapshots/<wave>/<project>.yaml`.
4. Запустить модель: открыть сессию, сказать «прогоняй задачу `<task-id>` на волне `<wave>`».
5. После прогонов: `python tools/aggregate.py` → `leaderboard.md`.
