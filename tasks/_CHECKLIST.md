# Чек-лист для участника

> Это список критериев, по которым будет оцениваться твоя работа.
> **Баллы не указаны** — это не для самооценки, а для понимания ожиданий.
> Судья оценивает по детальному рубрикатору, который тебе недоступен.
>
> Относись к этому как к acceptance criteria тикета в реальной работе:
> ты знаешь, что важно, но не знаешь точную шкалу грейдинга.

## Что важно

### Корректность
Задача решена, работает, соответствует постановке и критериям приёмки.
Для bugfix — найдены реальные баги, нет ложных срабатываний.
Для feature — фича работает, соответствует спеке.
Для refactor — структура улучшена, поведение сохранено.

### Сборка и тесты
Код компилируется. Тесты проходят. OpenSpec валиден (если проект использует OpenSpec).
Если что-то не запустилось — честно отметь в прогоне и объясни почему.

### Соблюдение правил проекта
Следуй `AGENTS.md` проекта:
- Один ViewModel (не создавай второй)
- Токены дизайн-системы (не inline `Color(0xFF...)` / `.dp` / `RoundedCornerShape(...)`)
- Миграции Room (не destructive в release)
- `safeDb` / обработка ошибок через Snackbar
- Сортировка списков по правилам проекта
- Архитектурные guardrails не нарушены

### Качество кода
Идиоматичный Kotlin/Compose:
- Корутины, StateFlow (не Callback Hell)
- Compose-идиомы (не императивный UI)
- Именование, читаемость
- Отсутствие мёртвого кода и избыточной вложенности
- Компактность (no boilerplate)

### Архитектура
- Изменения локализованы (не размазаны по кодовой базе)
- Нет leaky abstractions
- Новый код в правильных пакетах/слоях
- Для refactor — структура улучшена, не просто перемещено

### Полнота и самостоятельность
- Задача доведена до конца
- Документация/спеки обновлены если нужно
- Нет TODO без причины
- Если есть неоднозначности — запиши clarifying questions в прогон
  (никто не ответит, но судья оценит качество вопросов)
- Делай разумные предположения при неоднозначностях

### Заметки и объяснения
В прогоне заполни все секции:
- Что сделано — конкретно, без воды
- Что получилось хорошо
- Что не получилось / проблемы
- Trade-offs и решения
- Что бы улучшила
- Препятствия
- Вопросы по задаче (если есть)
