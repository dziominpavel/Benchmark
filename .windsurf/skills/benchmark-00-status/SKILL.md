Показывает текущий прогресс бенчмарка и подсказывает следующий шаг.

**Когда использовать:** в любой момент, когда хочешь узнать где ты в workflow
и что делать дальше.

## Шаги

1. **Прочитай состояние файловой системы:**
   - `tasks/` — сколько задач создано (по категориям)
   - `tasks/bugfix/*/golden-answer/` — сколько resolved (для bugfix)
   - `runs/` — сколько участников выполнили, по каким задачам
   - `verdicts/` — сколько вердиктов написано, какими судьями
   - `models.yaml` — сколько моделей зарегистрировано
   - `judges.yaml` — сколько судей в pool

2. **Определи текущий шаг:**
   - Шаг 1 (gen-task): если задач < ожидаемого
   - Шаг 2 (resolve): если есть bugfix без golden-answer
   - Шаг 3 (run): если есть участники без прогонов по всем задачам
   - Шаг 4 (judge): если есть прогоны без вердиктов
   - Шаг 5 (aggregate): если все вердикты готовы, leaderboard не обновлён

3. **Покажи:**
   ```
   Прогресс: Шаг N из 5

   ✓ Шаг 1: X задач создано (B-001, F-001, ...)
   ✓ Шаг 2: Y/Z bugfix resolved
   → Шаг 3: M/N участников выполнено
     ✓ claude-sonnet-4.5 (X прогонов)
     ✓ gpt-5 (X прогонов)
     ⬚ glm-4.6 — ожидает
   ⬚ Шаг 4: 0/K судей
   ⬚ Шаг 5: агрегация

   Дальше: @benchmark-NN-<action> <args>
   Осталось: X сессий
   ```

4. **Подскажи следующую команду** с точным синтаксисом.
