Диалоговая генерация задачи бенчмарка — по одной за раз.

**Когда использовать:** когда нужно создать новую задачу для бенчмарка.
Пользователь описывает тему → ты предлагаешь формулировку → он утверждает →
файл создаётся.

## Шаги

1. **Определи проект и категорию из запроса пользователя:**
   - Проект: GymProgress / VoiceMind / Wishlot
   - Категория: feature (F-NNN) / bugfix (B-NNN) / refactor (R-NNN) / review (V-NNN)
   - Если неясно — спроси

2. **Определи baseline_commit:**
   ```bash
   cd C:/projects/<Project>
   git rev-parse HEAD
   ```
   Это текущий HEAD проекта — зафиксируй его в task.md.

3. **Сгенерируй следующий ID:**
   - Посмотри `tasks/<cat>/` какие уже есть
   - Следующий: `<PREFIX>-<NNN>-<slug>` (например `B-002-workout-score-bugs`)

4. **Предложи формулировку задачи** (покажи пользователю):
   - Заголовок
   - Категория, сложность (1-3)
   - Описание (что сделать)
   - Контекст (модуль, файлы)
   - Критерии приёмки (конкретные, измеримые)
   - Ограничения
   - Для bugfix: НЕ перечисляй конкретные баги — только "найди N багов в модуле X"

5. **Жди утверждения:**
   - Пользователь: "добро" → создавай файл
   - Пользователь: "измени X" → обнови формулировку, снова спроси

6. **Создай файл:**
   ```
   tasks/<cat>/<ID>-<slug>/task.md
   ```
   По шаблону `tasks/_TEMPLATE.md` (если есть) или с front matter:
   ```yaml
   ---
   id: <ID>
   category: <feature|bugfix|refactor|review>
   project: <GymProgress|VoiceMind|Wishlot>
   baseline_commit: <hash>
   difficulty: <1|2|3>
   title: "<Заголовок>"
   ---
   ```

7. **Сообщи:**
   ```
   ✓ tasks/<cat>/<ID>-<slug>/task.md создан
   Дальше: @benchmark-01-gen-task (ещё задачу) или @benchmark-02-resolve <ID> (для bugfix)
   ```

## Правила

- **По одной задаче за раз** — не создавай несколько файлов сразу
- **Для bugfix** — НЕ перечисляй конкретные баги в task.md
- **baseline_commit** — всегда текущий HEAD проекта на момент генерации
- **Утверждение обязательно** — не создавай файл без "добро" от пользователя
