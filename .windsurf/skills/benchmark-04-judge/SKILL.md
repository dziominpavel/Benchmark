Запуск судьи — распределение судей по прогонам, изоляция чужих вердиктов,
подготовка промпта для оценки.

**Когда использовать:** когда все участники выполнили задачи и нужно
оценить прогоны. Пользователь вызывает один раз на каждого судью.

## Шаги

1. **Прими аргумент `<judge-id>`** — slug модели-судьи (например `qwen-3-coder`).

2. **Запусти распределение судей (первый раз):**
   ```bash
   python tools/assign_judges.py --distribute
   ```
   Это:
   - Анонимизирует прогоны (создаёт `.anon-copies/`)
   - Распределяет судей по прогонам (3 судьи на прогон)
   - Создаёт карту назначений
   - Если уже распределено — скрипт сообщит

3. **Запусти изоляцию вердиктов:**
   ```bash
   python tools/isolate.py hide-verdicts --task all --except <judge-id>
   ```
   Это скрывает чужие вердикты — судья не видит оценки других судей.

4. **Собери список прогонов для оценки:**
   - Прочитай `.anon-copies/` — анонимизированные прогоны
   - Для каждого: task-id, participant-anon-id, путь к run.md, diff.patch
   - Для bugfix: также golden-answer (он виден судье, не скрыт изоляцией вердиктов)

5. **Подготовь промпт для судьи:**
   ```
   Ты — судья. Прочитай judges/_PROMPT.md и judges/_RUBRIC.md.

   Сначала саморегистрируйся:
   python tools/assign_judges.py --add-judge <judge-id>

   Оцени прогоны: <N> прогонов (<M> участников × <K> задач)

   Для каждого прогона:
   1. Прочитай задачу: tasks/<cat>/<task-id>/task.md
   2. Прочитай анонимизированный прогон:
      .anon-copies/<task-id>/<participant-anon>/run.md
   3. Прочитай diff.patch (если есть)
   4. Для bugfix — прочитай golden answer:
      tasks/bugfix/<task-id>/golden-answer/answer.md
   5. Оцени по рубрикатору, каждый балл 0-5 с обоснованием
   6. Сохрани вердикт:
      verdicts/<task-id>/<participant-anon>/<judge-id>.md

   Прогоны для оценки:
   - <task-id-1> / Участник A / .anon-copies/...
   - <task-id-1> / Участник B / .anon-copies/...
   ...

   После всех — сообщи "готово, N вердиктов созданы".
   ```

6. **Покажи промпт пользователю:**
   ```
   ПРОМПТ ДЛЯ СУДЬИ (скопируй в новый чат с <judge-id>):

   <промпт>

   Когда судья закончит — напиши "готово".
   ```

7. **Жди подтверждения "готово" от пользователя.**

8. **Восстанови изоляцию вердиктов:**
   ```bash
   python tools/isolate.py restore-verdicts
   ```

9. **Проверь результат:**
   - Для каждого прогона: `verdicts/<task-id>/<participant-anon>/<judge-id>.md` существует?
   - Если нет — сообщи

10. **Сообщи:**
    ```
    ✓ <judge-id> выполнено (N/M вердиктов)
    Дальше: @benchmark-04-judge <next-judge> (следующий судья)
            или @benchmark-05-aggregate (все судьи готовы)
    ```

## Правила

- **Один судья за раз** — не запускай 2 судей одновременно
- **Изоляция вердиктов обязательна** — судья не должен видеть чужие оценки
- **Судья ≠ участник** — судья не оценивает свой собственный прогон
- **Restore-verdicts обязателен** — после каждого судьи восстанавливай
- **Golden answer виден судье** — изоляция вердиктов не скрывает golden,
  только вердикты других судей
