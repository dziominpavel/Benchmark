Агрегация вердиктов — запуск aggregate.py, показ leaderboard, подсветка disputed.

**Когда использовать:** когда все судьи завершили оценку прогонов.
Это финальный шаг прогона.

## Шаги

1. **Запусти агрегацию:**
   ```bash
   python tools/aggregate.py
   ```
   Это:
   - Читает все вердикты из `verdicts/`
   - Усредняет оценки судей по каждому прогону
   - Обновляет `leaderboard.md`

2. **Покажи leaderboard:**
   - Прочитай `leaderboard.md`
   - Покажи содержимое пользователю

3. **Подсвети disputed прогоны:**
   - Найди прогоны где разброс оценок судей > 2 (например 2.0, 4.0, 4.5)
   - Сообщи: "⚠ disputed: <task-id>/<participant> — разброс <X>"

4. **Сообщи:**
   ```
   ✓ Leaderboard обновлён
   <N> моделей в рейтинге
   <M> disputed прогонов — разбери вручную если нужно

   Прогон завершён!
   ```

## Правила

- **Запускай только когда все судьи готовы** — иначе агрегация будет неполной
- **Disputed — не ошибка** — просто подсвети, пользователь решит что делать
- **Leaderboard кумулятивный** — растёт со временем, не сбрасывается
