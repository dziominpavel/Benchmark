## Why

Текущая архитектура (change `simplify-architecture`) сократила количество сессий
с 184 до 13, но пользователь всё ещё вручную запускает python-скрипты
(`isolate.py`, `assign_judges.py`, `aggregate.py`), вручную редактирует
`judges.yaml` и `anonymization.yaml`, и копипастит промпты из чеклистов.
Цель — минимизировать ручной труд: работа пользователя должна состоять только
в открытии чатов с тестируемыми моделями и написании запросов. Все скриптовые
операции (изоляция, регистрация, агрегация, генерация промптов) должны
выполняться автоматически через skills Windsurf/Devin IDE.

Дополнительно: модели должны сами себя регистрировать в `models.yaml` и
`judges.yaml` при старте роли — пользователь не редактирует эти файлы руками.

## What Changes

- **NEW: Skills Windsurf/Devin** — набор skills в `.windsurf/skills/benchmark-*/`
  для автоматизации workflow: генерация задач, выполнение, оценка, агрегация,
  статус. Пользователь не запускает python-скрипты напрямую.
- **NEW: Саморегистрация моделей** — модель при старте роли (участник/судья)
  автоматически добавляет себя в `models.yaml` (и `judges.yaml` если судья).
  Пользователь не редактирует конфиги руками. `register_model.py` остаётся
  для ручного режима (опционально).
- **MODIFIED: Диалоговая генерация задач** — вместо батч-генерации "4 задачи
  за 1 сессию" — диалог: Devin помогает сформулировать задачу, пользователь
  утверждает, файл создаётся. По одной задаче за раз, не батч.
- **MODIFIED: Изоляция через skill** — skill автоматически запускает
  `isolate.py hide` перед сессией участника/судьи и `isolate.py restore`
  после. Пользователь не запускает isolate.py руками.
- **MODIFIED: Скрытие golden-answer** — `isolate.py hide` также скрывает
  `tasks/bugfix/*/golden-answer/` от участников. Это позволяет использовать
  любую модель как resolver без конфликта ролей (resolver может быть
  участником той же задачи, если golden answer скрыт).
- **MODIFIED: Агрегация через skill** — skill автоматически запускает
  `aggregate.py` после завершения оценки и показывает leaderboard.
- **MODIFIED: Промпт-генерация через skill** — skill готовит промпт для
  копипаста в чат с тестируемой моделью. Пользователь не запускает
  `generate_checklist.py` руками.

## Capabilities

### New Capabilities

- `ide-skills`: Skills для Windsurf/Devin IDE, автоматизирующие workflow
  бенчмарка: генерация задач, выполнение, оценка, агрегация, статус.
  Skills обёртывают python-скрипты, пользователь не запускает их напрямую.
- `self-registration`: Модели автоматически регистрируются в `models.yaml`
  и `judges.yaml` при старте роли. Пользователь не редактирует конфиги руками.

### Modified Capabilities

- `task-generation`: Диалоговая генерация задач (по одной, через skill)
  вместо батч-генерации "4 задачи за 1 сессию". Devin помогает
  сформулировать задачу, пользователь утверждает.
- `task-execution`: Изоляция и подготовка промпта выполняются skill'ом
  автоматически. Модель саморегистрируется при старте.
- `judging`: Изоляция вердиктов и подготовка промпта выполняются skill'ом
  автоматически. Модель-судья саморегистрируется в `judges.yaml`.
- `leaderboard`: Агрегация запускается skill'ом автоматически после
  завершения оценки.

## Impact

- **Новые файлы:** `.windsurf/skills/benchmark-*/SKILL.md` (6-8 skills)
- **Модифицируемые файлы:** `AGENTS.md` (саморегистрация вместо "попроси
  пользователя"), `generators/_PROMPT.md` (диалоговая генерация),
  `judges/_PROMPT.md` (саморегистрация судьи)
- **Скрипты:** `register_model.py` остаётся (опциональный ручной режим),
  `isolate.py` / `assign_judges.py` / `aggregate.py` вызываются из skills
- **Конфиги:** `models.yaml` и `judges.yaml` заполняются моделями
  автоматически, а не пользователем вручную
- **Совместимость:** skills в `.windsurf/skills/` (Windsurf/Devin IDE).
  Cross-agent: `.agents/skills/` также поддерживается Devin Desktop.
