## ADDED Requirements

### Requirement: Генерация задачи моделями-генераторами

Задачи на волну MUST генерироваться 2–3 моделями-генераторами независимо. Каждый генератор пишет черновик в `tasks/_drafts/<generator-id>/<task-id>.md` по шаблону `tasks/_TEMPLATE.md`. Генераторы не видят черновики друг друга.

#### Scenario: Независимые черновики

- **WHEN** 3 генератора получают тему «найди 5 багов в VoiceMind parser»
- **THEN** каждый пишет черновик в свою папку `tasks/_drafts/<generator-id>/`
- **AND** ни один генератор не имеет доступа к папкам других генераторов

#### Scenario: Генератор не знает ответа

- **WHEN** генератор формулирует задачу
- **THEN** он не решает задачу и не включает готовый ответ в черновик
- **AND** задача содержит только постановку, контекст, критерии приёмки, ограничения

### Requirement: Отбор финальной задачи

Пользователь MUST выбрать лучший черновик (или слить лучшие части из нескольких) и поместить финальную задачу в `tasks/<category>/<task-id>/task.md`. Финальная задача соответствует шаблону `tasks/_TEMPLATE.md`.

#### Scenario: Финальная задача в правильной папке

- **WHEN** задача F-001 отобрана
- **THEN** файл `tasks/feature/F-001-<slug>/task.md` существует
- **AND** его front matter содержит `id`, `title`, `category`, `difficulty`, `project` или `projects`, `tags`

### Requirement: Multi-project задача

Задача MAY затрагивать несколько проектов. В front matter поле `projects` (массив) MUST указывать список проектов если задача multi-project. Участник SHALL работать во всех указанных проектах, `diff.patch` может содержать изменения из нескольких.

#### Scenario: Два проекта в задаче

- **WHEN** задача имеет `projects: [voicemind, gymprogress]`
- **THEN** участник работает в `C:/projects/VoiceMind` и `C:/projects/GymProgress`
- **AND** `diff.patch` может содержать изменения из обоих

### Requirement: Категории задач

Задачи MUST относиться к одной из 4 категорий: `feature` (префикс F-NNN), `bugfix` (B-NNN), `refactor` (R-NNN), `review` (V-NNN). Префикс ID определяет категорию.

#### Scenario: ID определяет категорию

- **WHEN** задача имеет id `B-002-fix-parser-crash`
- **THEN** она находится в `tasks/bugfix/B-002-fix-parser-crash/task.md`
- **AND** её категория — `bugfix`

### Requirement: 8 задач на волну

Волна MUST содержать 8 задач: 2 feature, 2 bugfix, 2 refactor, 2 review. Если задача отбракована как нерабочая (после resolve или мета-анализа), она MAY быть заменена резервной.

#### Scenario: Состав волны

- **WHEN** волна `wave-01-2026Q3` готова к выполнению
- **THEN** в `tasks/` существует 2 задачи с префиксом F, 2 с B, 2 с R, 2 с V
- **AND** все отобраны для этой волны (указаны в `waves/<wave>/tasks.yaml` или эквиваленте)

### Requirement: Шаблон задачи

Каждая задача MUST следовать шаблону `tasks/_TEMPLATE.md`: front matter (`id`, `title`, `category`, `difficulty`, `project`/`projects`, `tags`, `estimated_minutes`) + секции «Контекст», «Постановка», «Критерии приёмки», «Ограничения», «Подсказки» (опц.), «Оценка».

#### Scenario: Обязательные секции

- **WHEN** задача `F-001` создана
- **THEN** файл содержит секции «Контекст», «Постановка», «Критерии приёмки»
- **AND** front matter содержит `id`, `category`, `project` или `projects`

### Requirement: Промпт для генератора

Система MUST иметь `generators/_PROMPT.md` — промпт для модели-генератора, аналогично `judges/_PROMPT.md` для судьи. Промпт описывает: что получает генератор (тема от пользователя + ссылки на спеки/docs проекта), что пишет (черновик по шаблону в `tasks/_drafts/<generator-id>/`), что НЕ делает (не решает задачу, не включает готовый ответ).

#### Scenario: Генератор читает промпт

- **WHEN** пользователь открывает сессию с моделью-генератором
- **THEN** модель читает `generators/_PROMPT.md` и следует протоколу
- **AND** пишет черновик в `tasks/_drafts/<generator-id>/<task-id>.md`

#### Scenario: Генератор не решает задачу

- **WHEN** генератор формулирует bugfix-задачу «найди 5 багов»
- **THEN** черновик содержит постановку и критерии, но НЕ содержит список конкретных багов
- **AND** генератор не выполняет поиск багов в коде проекта
