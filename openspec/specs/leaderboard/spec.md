## Purpose

Кумулятивная агрегация вердиктов в `leaderboard.md`. Не перегенерируется,
обновляется при добавлении новых прогонов/вердиктов. Разбивка по категориям
и задачам (не по волнам).

## Requirements

### Requirement: Агрегация через skill

Агрегация SHALL запускаться через skill `benchmark-05-aggregate` (не вручную).
Skill запускает `aggregate.py`, показывает `leaderboard.md`, подсвечивает
disputed прогоны. Пользователь не запускает `aggregate.py` напрямую.

#### Scenario: Skill запускает агрегацию

- **WHEN** пользователь говорит "aggregate"
- **THEN** skill `benchmark-05-aggregate` запускает `python tools/aggregate.py`
- **AND** показывает содержимое `leaderboard.md`
- **AND** подсвечивает disputed прогоны (разброс > 2)

#### Scenario: Пользователь не запускает python

- **WHEN** пользователь хочет обновить leaderboard
- **THEN** он говорит "aggregate" (не запускает `python tools/aggregate.py`)
- **AND** skill вызывает скрипт
- **AND** пользователь не видит python-команды

### Requirement: Кумулятивная агрегация

`aggregate.py` MUST кумулятивно обновлять `leaderboard.md` при каждом запуске.
Новые прогоны/вердикты добавляются к существующим. Не перегенерирует с нуля.
Модели, добавленные инкрементально, включаются в общий рейтинг.

#### Scenario: Новая модель в рейтинге

- **WHEN** `model-F` добавлена инкрементально, 3 судьи оценили её прогоны
- **AND** `aggregate.py` запущен
- **THEN** `leaderboard.md` содержит `model-F` в общем рейтинге
- **AND** существующие модели A..E не затронуты

#### Scenario: Нет вердиктов

- **WHEN** в `verdicts/` нет файлов
- **THEN** `aggregate.py` генерирует `leaderboard.md` с сообщением «Нет вердиктов»

### Requirement: Деанонимизация (глобальная)

`aggregate.py` MUST читать `anonymization.yaml` в корне репозитория (глобальная
карта) и заменять «Участник X» на реальный `model_id` в leaderboard.

#### Scenario: Деанонимизация

- **WHEN** вердикт судит «Участник A» с баллом 4.5
- **AND** `anonymization.yaml` содержит `claude-sonnet-4.5 → "Участник A"`
- **THEN** leaderboard показывает `claude-sonnet-4.5 | 4.5`

### Requirement: Структура leaderboard (кумулятивный)

`leaderboard.md` MUST содержать: общий рейтинг моделей (средний нормализованный
балл), разбивку по категориям (F/B/R/V), разбивку по задачам. НЕТ разбивки по
волнам. Каждая таблица — Markdown.

#### Scenario: Общий рейтинг

- **WHEN** leaderboard сгенерирован
- **THEN** секция «Общий рейтинг моделей» содержит таблицу: Модель | Прогонов | Средний балл | F | B | R | V

#### Scenario: По задачам

- **WHEN** leaderboard сгенерирован
- **THEN** секция «По задачам» содержит подтаблицу для каждой задачи: Модель | Балл | Судьи

### Requirement: Кумулятивное обновление

`aggregate.py` MUST обновлять `leaderboard.md` при каждом запуске, добавляя
новые данные к существующим. Не перезаписывает с нуля. Ручные правки
`leaderboard.md` не сохраняются — файл не редактируется руками.

#### Scenario: Обновление

- **WHEN** `aggregate.py` запущен после новых вердиктов
- **THEN** `leaderboard.md` обновлён с новыми данными
- **AND** существующие данные сохранены

### Requirement: Карта назначений (по задачам)

`aggregate.py` MUST генерировать таблицу назначений: для каждого прогона
(задача × участник) — 3 судьи с их оценками и итоговой медианой.

#### Scenario: Карта в leaderboard

- **WHEN** leaderboard сгенерирован
- **THEN** секция «Карта назначений» содержит таблицу: Задача | Участник | Судья 1 | Судья 2 | Судья 3 | Медиана
- **AND** для спорных прогонов строка помечена `⚠ disputed`

### Requirement: Спорные прогоны в leaderboard

Прогон с разбросом > 2 между судьями MUST помечаться `disputed` в leaderboard.
Пользователь разбирает disputed прогоны вручную.

#### Scenario: Disputed прогон

- **WHEN** судьи оценили прогон с баллами 2.0, 4.0, 4.5
- **THEN** разброс = 2.5 (> 2) → прогон помечен `⚠ disputed`

### Requirement: Качество судей (согласованность)

`aggregate.py` MUST генерировать таблицу качества судей: для каждого судьи —
количество вердиктов, средний балл, разброс. Высокий разброс = низкая
согласованность.

#### Scenario: Качество судей в leaderboard

- **WHEN** leaderboard сгенерирован
- **THEN** секция «Качество судей» содержит таблицу: Судья | Вердиктов | Средний балл | Разброс

### Requirement: Только stdlib Python

`aggregate.py` MUST использовать только стандартную библиотеку Python.
Никаких внешних зависимостей.

#### Scenario: Запуск без зависимостей

- **WHEN** `python tools/aggregate.py` запущен
- **THEN** скрипт выполняется без установки внешних пакетов
