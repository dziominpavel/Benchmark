# Workflow — пошаговая инструкция прогона

> Эта инструкция описывает минимальный прогон: 3 участника, 3 судьи,
> 2 задачи (1 bugfix + 1 feature). 8 чатов, ~1.5-2 часа.
>
> Для большего прогона — просто повторяй шаги 3 и 4 для дополнительных
> участников/судей.

## Чаты и их назначение

| Чат # | Модель           | Назначение                          | Когда открыть         |
|-------|------------------|-------------------------------------|-----------------------|
| Чат 0 | IDE-агент        | Руление workflow, все skills        | Первым, не закрывать  |
| Чат 1 | Resolver-модель  | Golden answer для B-001             | На шаге 2             |
| Чат 2 | Claude           | Участник (2 задачи, батч)           | На шаге 3             |
| Чат 3 | GPT-5            | Участник (2 задачи, батч)           | На шаге 3             |
| Чат 4 | GLM-4.6          | Участник (2 задачи, батч)           | На шаге 3             |
| Чат 5 | Qwen-3-coder     | Судья (6 прогонов)                  | На шаге 4             |
| Чат 6 | Llama-4-70b      | Судья (6 прогонов)                  | На шаге 4             |
| Чат 7 | Mistral-Large-2  | Судья (6 прогонов)                  | На шаге 4             |

**Паттерн:** IDE → MODEL → IDE → MODEL → ... → IDE
- В IDE-чате вызываешь skill, получаешь промпт
- В чате с моделью вставляешь промпт, ждёшь результат
- Возвращаешься в IDE-чат, пишешь "готово"
- **Никогда не открывай 2 чата с моделями одновременно** — изоляция работает
  только для одной модели за раз

## Skills — пронумерованы по порядку

| Skill                    | Шаг | Что делает                          |
|--------------------------|-----|-------------------------------------|
| `@benchmark-00-status`   | —   | Показывает прогресс, подсказывает   |
|                          |     | следующий шаг. Доступен ВСЕГДА.     |
| `@benchmark-01-gen-task` | 1   | Диалоговая генерация задачи         |
| `@benchmark-02-resolve`  | 2   | Golden answer для bugfix            |
| `@benchmark-03-run`      | 3   | Запуск участника (изоляция+промпт)  |
| `@benchmark-04-judge`    | 4   | Запуск судьи (изоляция+промпт)      |
| `@benchmark-05-aggregate`| 5   | Агрегация → leaderboard             |
| `@benchmark-06-stale`    | —   | Опционально: проверка устаревания   |

## ЛЮБОЙ МОМЕНТ: проверить статус

**Чат 0 (IDE-агент):**

```
@benchmark-00-status
```

IDE-агент показывает:
- На каком шаге ты сейчас
- Что сделано (задачи, resolve, участники, судьи)
- Что осталось
- Какую команду вызвать следующей
- Сколько сессий осталось

Пример вывода:
```
Прогресс: Шаг 3 из 5

✓ Шаг 1: 2 задачи созданы (B-001, F-001)
✓ Шаг 2: B-001 resolved (golden answer готов)
→ Шаг 3: 2/3 участников выполнено
  ✓ claude-sonnet-4.5 (2 прогона)
  ✓ gpt-5 (2 прогона)
  ⬚ glm-4.6 — ожидает
⬚ Шаг 4: 0/3 судей
⬚ Шаг 5: агрегация

Дальше: @benchmark-03-run glm-4.6
Осталось: 1 участник + 3 судьи + агрегация = 5 сессий
```

## Инкрементальное добавление новой модели (после первого прогона)

### Добавить участника

1. `@benchmark-03-run model-F` → вставь промпт в новый чат с model-F → `готово`
2. `@benchmark-04-judge qwen-3-coder` → оценить только новые прогоны
3. `@benchmark-04-judge llama-4-70b` → оценить только новые прогоны
4. `@benchmark-04-judge mistral-large-2` → оценить только новые прогоны
5. `@benchmark-05-aggregate` → обновлённый leaderboard

### Добавить задачу

1. `@benchmark-01-gen-task` → создать новую задачу
2. `@benchmark-02-resolve <new-task-id>` → если bugfix
3. Для каждого участника: `@benchmark-03-run <model>` → выполнить новую задачу
4. Для каждого судьи: `@benchmark-04-judge <judge>` → оценить новые прогоны
5. `@benchmark-05-aggregate` → обновлённый leaderboard

## Правила

1. **Один чат с моделью за раз** — изоляция работает только для одной модели
2. **Всегда возвращайся в Чат 0** после работы с моделью — пиши "готово"
3. **Не открывай чаты с моделями заранее** — открывай когда skill дал промпт
4. **Запутался → `@benchmark-00-status`** — покажет где ты и что дальше
5. **Resolver ≠ участник той же задачи** — если не уверен, выбери модель
   не из участников (golden answer скрыт изоляцией, но лучше перестраховаться)
6. **Судья ≠ участник** — судья не оценивает свой собственный прогон
